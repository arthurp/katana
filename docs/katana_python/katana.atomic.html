Atomic Operations (katana.atomic)¶

This module provides atomic values and atomic operations on numpy arrays. The atomic values are instantiated for several element types, but cannot support custom element types. Atomic operations on numpy arrays support integer and floating point arrays.

The instantiations are:

  • np.uint64 (with C element type uint64_t)

  • int (with C element type int64_t)

  • np.uint32 (with C element type uint32_t)

  • np.int32 (with C element type int32_t)

  • float (with C element type double)

  • np.float32 (with C element type float)

The instantiation is selected by indexing the type, for example GAccumulator[int] or GReduceMin[np.uint64].

class katana.atomic.GAccumulator¶

A reducer object that can updated with new values and combines the results efficiently using addition.

This class can be passed into numba compiled code and it’s methods can be used from there.

(This class wraps the underlying C++ type GAccumulator.)

__init__(self, v=None)¶

Create a GAccumulator. If v is provided and is not None, then initialize the reducer to v.

get_local(self)¶

Get a sub-result of the reducers operation. This is generally the reduced value for this thread.

reduce(self)¶

Get the current value of the reducer.

This must only be called from single threaded code.

reset(self)¶

Reset the reducer to zero.

This must only be called from single threaded code.

update(self, v)¶

Update this reducer with v performing the operation.

katana.atomic.atomic_add(ary, i, v)[source]¶

Atomically, perform ary[i] += v and return the previous value of ary[i].

i must be a simple index for a single element of ary. Broadcasting and vector operations are not supported.

This should be used from numba compiled code.

katana.atomic.atomic_max(ary, i, v)[source]¶

Atomically, perform ary[i] = max(ary[i], v) and return the previous value of ary[i]. This operation does not support floating-point values.

i must be a simple index for a single element of ary. Broadcasting and vector operations are not supported.

This should be used from numba compiled code.
